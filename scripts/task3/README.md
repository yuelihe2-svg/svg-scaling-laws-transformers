# Part 3 — μP scaling (`mup`)

This folder implements a **μP** variant of the Task 2 transformer (`model_mup.py`) with:

- **`MuReadout`** for the LM head (`readout_zero_init=True`)
- **Causal attention logits scaled by `1 / d_head`** (not `1/sqrt(d_head)`)
- **Zero-init query projection** (`wq`) after `mup.init`, per `mup` README
- **`mup.set_base_shapes(model, base, delta)`** with **default** `base=tiny`, `delta=small`
- **`MuAdamW`**; the training loop scales **every** optimizer group’s LR by the **same** factor
  `lr_scalar / peak_lr` so μP’s per-width `initial_lr` ratios are preserved (do **not** assign one global LR to all groups).

## Presets vs Part 2

`config_presets.py` defines **`MU_PRESETS`**: all widths use **`n_layers=12`** and **`n_heads=8`** so the module tree matches across sizes for `set_base_shapes`. **Part 2** uses **different depths** per name; **XL** parameter count matches Part 2 for the same `vocab_size` and `block_size`, but smaller names do **not** match Part 2 param counts. **State this clearly** when overlaying SP vs μP scaling curves.

## Commands (from repo root)

LR sweep on smallest μP model:

```bash
python -m scripts.task3.lr_sweep_mup \
  --train-jsonl data/processed/train.jsonl \
  --val-jsonl data/processed/val.jsonl \
  --spm-model data/processed/spm.model \
  --lrs 1e-4 3e-4 1e-3 3e-3 1e-2 3e-2 1e-1 \
  --out-csv outputs/task3/lr_sweep_mup.csv
```

Train one size (use best LR from sweep for all sizes):

```bash
python -m scripts.task3.train_mup \
  --train-jsonl data/processed/train.jsonl \
  --val-jsonl data/processed/val.jsonl \
  --spm-model data/processed/spm.model \
  --preset medium --lr <BEST> \
  --tokens-per-batch 32768 --block-size 512 \
  --out-dir outputs/task3/medium
```
